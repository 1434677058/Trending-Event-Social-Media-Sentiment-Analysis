# 热点事件社交媒体舆情分析系统

一个基于 Flask 的本地舆情分析系统，聚焦社交媒体评论场景，完成从评论采集、文本预处理、情感分析、主题挖掘到可视化展示的完整流程。当前版本以 B 站评论为主要数据源，同时保留本地文本导入能力用于扩展测试。

## 项目定位

本项目面向“热点事件评论舆情分析”场景，目标是帮助用户快速完成以下工作：

- 围绕指定热点关键词或 B 站视频抓取公开一级评论
- 对评论文本进行清洗、去重和分词
- 基于 SnowNLP 进行情感倾向分析
- 基于 LDA 进行主题挖掘和主题解释
- 通过系统概览页和分析页展示舆情趋势、热点排行和主题结果

项目更强调“本地可运行、流程完整、展示直观”，适合课程设计、毕业设计和小型研究型演示场景。

## 核心功能

### 1. 数据采集

- 支持输入 B 站视频链接、`BV` 号、`av` 号或关键词
- 支持通过关键词搜索相关热门视频，再抓取视频下公开一级评论
- 支持保存并读取 B 站 Cookie 配置
- 支持采集任务管理、进度查看和历史记录回看

### 2. 文本预处理

- 去除 HTML、链接、话题、特殊符号等噪声信息
- 使用 `jieba` 完成中文分词
- 使用停用词过滤无意义词项
- 对任务内评论文本进行去重

### 3. 情感分析

- 使用 `SnowNLP` 对清洗后的评论进行情感评分
- 根据分数区间划分为正面、中性、负面
- 输出情感统计、评分分布和评论明细

### 4. 主题挖掘

- 使用 `scikit-learn` 中的 `LatentDirichletAllocation` 实现 LDA 主题建模
- 自动生成主题关键词、主题名称、主题解释和代表文本
- 对低信息密度词进行二次过滤，提升主题结果可读性

### 5. 数据可视化

- 系统概览：数据总量、热点事件数、情感占比
- 情感趋势：按日期聚合后的情感走势
- 热点事件排行：基于评论热度与点赞表现的综合排行
- 词云、热门话题、情感占比图、共现网络图等可视化结果

## 项目特点

- 端到端流程完整：从采集到分析到展示都在同一套系统内完成
- 面向中文舆情文本：预处理、分词与情感分析都围绕中文场景设计
- 可解释性较强：主题挖掘结果不仅有关键词，还有说明文本和代表样本
- 本地部署门槛低：使用 Flask + SQLite，无需单独部署数据库服务
- 适合演示和答辩：界面结构清晰，方便展示功能模块和分析结果

## 技术栈

- 后端：Flask
- 数据库：SQLite
- 数据处理：pandas、numpy
- 分词：jieba
- 情感分析：SnowNLP
- 主题建模：scikit-learn LDA
- 前端：HTML、CSS、原生 JavaScript、ECharts
- 采集相关：requests、beautifulsoup4、xhshow、playwright

## 运行环境

- Python 3.10 及以上
- Windows / macOS / Linux 均可，本项目最初在 Windows 环境下开发

## 安装与运行

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 初始化数据库

可选方式一：

```bash
python database.py
```

可选方式二：直接启动应用，程序会自动执行建表逻辑。

### 3. 启动项目

```bash
python app.py
```

默认访问地址：

```text
http://127.0.0.1:5001/
```

## 数据库说明

本公开仓库不包含任何真实数据库数据、Cookie 配置或运行日志，只保留数据库结构与初始化代码。

核心数据表如下：

- `crawl_tasks`：采集任务表
- `weibo_posts`：评论文本表
- `sentiment_results`：情感分析结果表
- `topic_results`：主题分析结果表

完整建表语句见 `schema.sql`。

## 隐私与仓库清理说明

为了避免上传个人隐私和本地运行痕迹，仓库已忽略以下内容：

- `data/` 目录下的本地数据库、Cookie 配置和导入数据
- 日志文件、PID 文件、任务运行文件
- 本地文档产物、章节素材、渲染图片
- 本地辅助目录和自动化工具缓存

因此你克隆本仓库后，需要自行在本地重新生成数据库，并按需配置 B 站 Cookie。
